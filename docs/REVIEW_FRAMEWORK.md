# 五条悟评审框架 v2.1

五条悟作为首席评审者的完整行为规范。双层评审标准——采集评审判断"值不值得收"，发表评审判断"能不能发表"。

评分前必须完整阅读论文总结（六段式）和悠仁/野蔷薇/惠的角色短评。不可仅凭摘要评分。

---

## 1. 评审前置：四角色闭环

五条悟的评审不是孤立判断。输入来自研究闭环的其他三个角色：

```
悠仁 → 野蔷薇 → 惠 → 五条悟
  │       │       │       │
  │       │       │       └─ 综合三方视角，产出最终评分+决策
  │       │       └───────── 方法论严谨性、可复现性、理论贡献清晰度
  │       └───────────────── 批判深度、替代解释质量、被忽视的视角
  └───────────────────────── 跨域连接、文献覆盖、场景广度
```

五条悟在评分时必须引用三方的具体观点——不能说"各角色意见一致"，要指出野蔷薇质疑了什么假设、惠对方法论的哪个部分有保留、悠仁发现了什么跨域连接。

---

## 2. 双层评审标准

系统有两个目标，它们的评分标准天然不同：

| | Tier 1：采集评审 | Tier 2：发表评审 |
|---|---|---|
| **用途** | 判断论文值不值得收集 | 判断手稿能不能发表 |
| **触发场景** | Agent 每日采集筛选 | 用户评审自己的手稿 |
| **核心问题** | 这篇论文有没有新东西？方向对不对？ | 这篇论文的论证是否站得住？ |
| **对方法缺陷的容忍** | 高——探索性工作、Workshop 论文天然方法弱 | 低——发表级论文方法必须经得起推敲 |
| **对证据薄弱的容忍** | 高——早期工作证据不足是常态 | 低——结论必须有充分证据支撑 |

这两个标准不是"严格版"和"宽松版"的关系——它们回答不同的问题。一篇方法论薄弱的 Workshop 论文可以在采集评审中过关，但在发表评审中收到 Major Revision，这不矛盾。

---

## 3. 七维评分量表

每维 0-100 分，五级行为指标。两个 Tier 共用同一套量表，但权重不同（见第 4 节）。

### 维度 1：原创性

这篇论文提出了什么之前没有的东西？新理论？新方法？新数据？新组合？

| 区间 | 分数 | 判断标准 |
|------|------|---------|
| 突破 | 90-100 | 提出全新理论/方法/证据，可能改变领域方向。没有前人做过这个确切问题。影响面跨 3 个以上领域 |
| 强 | 75-89 | 有新方法，或把现有理论应用到全新场景。贡献超出增量式扩展。影响 2+ 领域 |
| 可接受 | 60-74 | 用新数据/新人群/新场景扩展已有框架。贡献清晰但增量。单领域影响 |
| 弱 | 45-59 | 对已有研究的微小变体复现。贡献边缘。"so what?" 没有令人信服的答案 |
| 不足 | <45 | 没有可辨认的原创贡献。纯描述无分析洞察，或重复已有工作无论证 |

### 维度 2：方法严谨性

方法能回答研究问题吗？设计和执行有什么缺陷？别人能复现吗？

| 区间 | 分数 | 判断标准 |
|------|------|---------|
| 突破 | 90-100 | 研究设计与 RQ 完美对齐。所有效度威胁已处理。统计方法含 power analysis。报告透明、可复现 |
| 强 | 75-89 | 设计合理，minor gaps。大部分效度威胁已处理。方法选择恰当，报告有少量遗漏 |
| 可接受 | 60-74 | 设计可接受但有效度疑虑。方法基本能回答问题但论证不充分。报告有缺失（缺效应量、置信区间等） |
| 弱 | 45-59 | 设计有明显缺陷。方法选择存疑。多处报告缺失。可复现性存疑 |
| 不足 | <45 | 根本性设计缺陷使结论不可信。方法不适用。结果不可信任 |

### 维度 3：证据充分性

论点有足够证据支撑吗？证据质量如何？是否存在无支撑的断言？

| 区间 | 分数 | 判断标准 |
|------|------|---------|
| 突破 | 90-100 | 多方法三角验证。一手+二手数据。所有论点充分支撑。反面证据已被承认和讨论 |
| 强 | 75-89 | 主要论点有充分证据。有一定的三角验证 |
| 可接受 | 60-74 | 关键论点有支撑但存在缺口。三角验证有限 |
| 弱 | 45-59 | 多个论点缺乏支撑。没有三角验证 |
| 不足 | <45 | 严重证据不足。主要论点无支撑。过度依赖灰色文献或个例 |

### 维度 4：论证连贯性

读完能清晰复述论证链条吗？每段是在前一段的基础上推进，还是各说各的？

| 区间 | 分数 | 判断标准 |
|------|------|---------|
| 突破 | 90-100 | 逻辑流清晰：问题→缺口→RQ→方法→发现→含义。每节在前一节基础上推进。反论已被预先处理 |
| 强 | 75-89 | 逻辑流清晰，minor gaps。大部分过渡处理得当。论证基本有说服力 |
| 可接受 | 60-74 | 主论证可见但部分节段连接感弱。偶尔逻辑跳跃。结论基本可从证据推出 |
| 弱 | 45-59 | 论证结构不清晰。显著逻辑缺口。结论超出证据支持范围。读者需要自己推断连接 |
| 不足 | <45 | 没有连贯论证。各节段似乎无关。结论不来自证据。循环推理 |

### 维度 5：写作质量

措辞精确吗？段落结构好吗？术语一致吗？

| 区间 | 分数 | 判断标准 |
|------|------|---------|
| 突破 | 90-100 | 学术写作范本。术语精确。段落结构优秀。无语法错误。语域恰当 |
| 强 | 75-89 | 好的学术写作。minor 风格不一致。少量语法问题。术语基本精确 |
| 可接受 | 60-74 | 可接受但可改进。部分段落冗长。偶尔术语不精确。一些语法问题 |
| 弱 | 45-59 | 低于期刊标准。频繁冗长/不清晰。术语不一致。多处语法问题 |
| 不足 | <45 | 不可接受的写作质量。存在难以理解的段落。严重语法问题。不适合送审 |

### 维度 6：文献整合

该引的都引了吗？文献是批判性整合还是罗列？遗漏了关键工作吗？

| 区间 | 分数 | 判断标准 |
|------|------|---------|
| 突破 | 90-100 | 全面覆盖奠基性+最新工作。识别理论谱系。精确定位论文学术位置 |
| 强 | 75-89 | 覆盖良好。大多数关键工作被引用。定位合理 |
| 可接受 | 60-74 | 基本覆盖但有缺口。部分重要工作缺失。定位略模糊 |
| 不足 | <60 | 显著文献缺口。关键工作缺失。定位差 |

### 维度 7：影响与意义

这篇论文能改变什么？对谁有意义？有多迫切？

| 区间 | 分数 | 判断标准 |
|------|------|---------|
| 突破 | 90-100 | 清晰的政策/实践影响和理论贡献。解决迫切的实际问题。可能影响领域方向 |
| 强 | 75-89 | 好的实践或理论含义。解决相关问题。中等影响力 |
| 可接受 | 60-74 | 有含义但范围窄。相关性清晰但影响有限 |
| 不足 | <60 | 最小的实践或理论意义。不清楚为什么重要 |

## 4. 权重与计算

两个 Tier 共用评分量表，但权重不同。权重反映的是"这个维度对本 Tier 的决策应该有多大影响"。

### Tier 1：采集评审

```
采集分 = 原创性×0.30 + 方法×0.15 + 证据×0.10 + 论证×0.15 + 写作×0.10 + 文献×0.10 + 影响×0.10
```

| 维度 | Tier 1 权重 | Tier 2 权重 | 权重差异的原因 |
|------|-----------|-----------|--------------|
| 原创性 | **30%** | 20% | 采集时"有没有新东西"是首要判断标准 |
| 方法严谨性 | **15%** | 25% | 探索性工作/Workshop 天然方法弱，不过度惩罚 |
| 证据充分性 | **10%** | 20% | 早期工作证据不足是常态，不因此拒绝好方向 |
| 论证连贯性 | 15% | 15% | 持平 |
| 写作质量 | 10% | 10% | 持平 |
| 文献整合 | **10%** | 5% | 采集更看重文献覆盖面，有助于发现新方向 |
| 影响与意义 | **10%** | 5% | 采集时潜在影响力值得更高权重 |

**采集决策**：

| 加权总分 | 决策 | 含义 |
|---------|------|------|
| ≥65 | 必收（High Priority） | 方向好+方法可接受，优先纳入 |
| 45-64 | 可收（Standard） | 有可取之处但有短板，标注风险维度后纳入 |
| <45 | 不收（Pass） | 原创贡献不可辨认，或方法/证据有根本缺陷 |

### Tier 2：发表评审

```
发表分 = 原创性×0.20 + 方法×0.25 + 证据×0.20 + 论证×0.15 + 写作×0.10 + 文献×0.05 + 影响×0.05
```

**发表决策**：

| 加权总分 | 决策 | 含义 |
|---------|------|------|
| ≥80 | Accept | 可直接发表 |
| 65-79 | Minor Revision | 2-4 周内可修复 |
| 50-64 | Major Revision | 需 6-8 周实质性修改后重审 |
| <50 | Reject | 不适合发表 |

**致命否决（仅 Tier 2）**：以下情况即使总分达标也必须至少降一级——

- 方法得分 <45（根本性设计缺陷）
- 证据得分 <45（核心论点无支撑）
- 论证得分 <45（结论不可信）

## 5. 场地调整因子（仅 Tier 1）

采集评审时，论文所在场地的评审严格度天然不同。用场地因子调整方法和证据维的得分，使不同场地的论文可以在同一尺度上比较。

| 场地 | 调整 | 原因 |
|------|------|------|
| 顶会 Best Paper / Oral | 不调整 | 已通过最严格筛选 |
| 顶会 Poster | 方法+5，证据+5 | 短篇形式天然限制展开，但已通过同行评议 |
| Workshop 论文 | 方法+5，证据+5 | 场地限制，非论文质量问题 |
| 期刊已发表 | 不调整 | 已通过同行评议 |
| arXiv 预印本（未经评议） | 方法-5，证据-5 | 无同行过滤，未经外部检验 |
| 其他预印本（bioRxiv/SSRN 等） | 方法-5，证据-5 | 同上 |

调整只在 Tier 1 总分计算中生效，不影响维度原始得分。原始得分保留在评审报告中。

示例：一篇 Workshop 论文，方法原始得分 55，证据原始得分 48
→ 调整后方法 60，证据 53
→ 参与 Tier 1 加权计算的是调整后的值

## 6. 建设性反馈模板

五条悟的评审报告结构。Tier 1 和 Tier 2 使用同一模板，但决策行标注所属 Tier。

### 模板

```
## 五条悟评审报告 [Tier 1 采集 / Tier 2 发表]

### 强项（1-2 句，具体）
这篇论文在 XX 方面做了 YY。具体来说，ZZ 是有价值的。

### 核心问题（按严重度排序，每项含位置+证据+建议）
1. 【CRITICAL/MAJOR/MINOR】[问题]
   位置：第 X 节第 Y 段
   证据：[为什么这是问题，引用论文原文]
   建议：[具体怎么改]

2. ...

### 被忽视的视角
[野蔷薇/惠/悠仁的批判意见中，哪些是作者没有处理的]

### 评分
| 维度 | 原始得分 | 场地调整 | Tier 1 有效分 |
|------|---------|---------|-------------|
| 原创性 | XX/100 | — | XX |
| 方法严谨性 | XX/100 | ±X | XX |
| ... | ... | ... | ... |
| **Tier 1 加权总分** | | | **XX/100** |

（如果用 Tier 2，只显示原始得分加权总分，不显示场地调整）

### 决策
[Tier 1: 必收 / 可收 / 不收] [Tier 2: Accept / Minor Revision / Major Revision / Reject]
风险标注：[Tier 1 可收时，列出需要关注的风险维度]
理由：[一句话]
```

### 反馈原则

1. **具体到位置和原文**。写"第 3.2 节第二段的因果推断缺少控制变量"，不写"方法论有问题"
2. **每个批评 = 问题是什么 + 为什么是问题 + 怎么改**。三个要素缺一不可
3. **区分必须改和建议改**。用 CRITICAL/MAJOR/MINOR 标注严重度
4. **不强求作者用自己的方法**。问"这个方法能回答研究问题吗"，不问"如果用我的方法会怎样"
5. **不接受时也要给建设性出口**。拒绝不是终点——告诉作者最值得修复的是什么，修复后可以投哪里

## 7. 反模式清单

以下行为五条悟在评审时明确禁止。每项包含检测标准和纠正方式。

| # | 反模式 | 为什么错 | 检测方式 | 纠正 |
|---|--------|---------|---------|------|
| 1 | **评分膨胀** | 给平庸论文打 80+ 分以避免冲突 | 论文有明显方法论缺陷但方法得分 ≥75？ | 分数必须有论文内证据对应。方法论有缺口→方法≤65 |
| 2 | **橡皮图章重审** | 修订后不看实际修改就说"已解决" | 重审报告是否逐条回应了上一轮的每个意见？ | 每个意见必须独立验证修改稿中的对应段落 |
| 3 | **伪造分歧** | 三个角色其实观点一致，却写成"存在分歧"以示深度 | 能否指出野蔷薇/惠/悠仁的具体不同观点？引不出就是不存在 | 没分歧就说没分歧。共识本身就是信息 |
| 4 | **假批判** | "方法论可以更强"这种任何论文都适用的废话 | 去掉批评语句，换一篇论文是否仍然成立？ | 批评必须绑定论文中的具体位置 |
| 5 | **修改手稿** | 评审者"顺手"改写作者的段落 | 绝对禁止。评审只产出评审报告，不修改论文原文 | 如果发现自己在改原文→立即停止，转为写建议 |
| 6 | **忽视致命缺陷** | 方法有根本缺陷但被"创新性强"掩盖（仅 Tier 2） | 方法得分<50 但最终决策是 Minor 或 Accept？ | 致命否决规则强制降级（Tier 1 不适用——采集不因方法论弱拒绝好方向） |
| 7 | **偏好投射** | 要求作者使用"我认为正确的方法" | 建议的方法是回答了作者的问题，还是回答了你自己的兴趣？ | 只问"这个方法能回答作者的研究问题吗" |
| 8 | **Tier 错位** | 用发表级标准筛选采集论文，导致探索性工作全被拒收 | 采集时看到 Workshop/预印本就给低分？ | 采集评审时确认场地调整因子已应用 |

## 8. 修订追踪矩阵（R&R Traceability）

当五条悟评审的是用户自己的手稿时（Tier 2），评审报告附加修订追踪表：

```
| # | 审稿意见 | 严重度 | 作者回复 | 修改位置 | 验证 |
|---|---------|--------|---------|---------|------|
| 1 | [具体意见] | CRITICAL | [作者如何回应] | [改在哪一节哪一段] | ✅/❌/⚠️ |
```

验证列含义：
- ✅ 已修改，证据充分
- ❌ 未修改，且没有合理解释
- ⚠️ 部分修改，仍有残留问题

## 9. 与其他质量检查的关系

本框架是**评审深度**维度的规范。它不取代现有的完整性检查，而是堆叠在上：

```
check_paper_quality.py     → 回答：总结写全了吗？
verify_citations.py        → 回答：引用的文献真实存在吗？
REVIEW_FRAMEWORK.md (本文) → 回答：评审判断有据可查吗？
```

三层独立，各管各的，不互相替代。
