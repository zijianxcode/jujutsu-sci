# 内容采集规则

## 最小采集单元

采集的最小单元是一篇论文。每篇论文对应一个目录，目录里只有一个完整论文总结：

```text
records/YYYY/MM/DD/HH/<paper-key>/论文总结.md
```

这样可以避免同一篇论文在论文总结、角色长文、团队讨论和升级迭代里重复出现完整摘要。

## 文件职责

`论文总结.md` 是事实源，负责完整记录论文信息和六段式分析。

`<角色>-能力进化.md` 是判断源，负责记录某个角色对这篇论文的评分、批判、关联和下一步动作。

`团队讨论.md`、`深度讨论.md`、`升级迭代.md` 不作为默认采集产物。只有当用户明确要求一次团队讨论或阶段评审时才生成。

**周会讨论**：每周一次团队讨论，生成 `团队讨论.md` 或 `周会讨论.md`。日会记录已取消。

## 必填信息

`论文总结.md` 必须写明：

- 论文标题
- 作者
- 来源、会议、期刊、arXiv、DOI 或 URL
- 日期
- 领域标签
- 综合评分：新采集统一使用 10 分制，例如 `8.5/10`

正文保持六段式：

1. 研究什么？
2. 为什么研究？
3. 别人做过什么？
4. 作者怎么研究的？
5. 发现了什么？
6. 价值和不足？

## 去冗余规则

同一篇论文已有 `<paper-key>/论文总结.md` 时，只更新该文件，不新建第二个总结。

角色短评不得复制 `论文总结.md` 的完整内容。它只保留角色自己的判断。

`*-论文总结.md` 这类带角色前缀的文件不再作为新采集格式使用。

## 领域调研策略

角色能力进化文档（`<角色>-能力进化.md`）应当记录在采集过程中发现的方法论经验，用于沉淀检索策略和数据库知识。策略内容统一放在一个名为 `## 领域调研策略` 的 section 中。

### 推荐策略内容

- **有效数据库**：哪些数据库/平台在本方向产出质量较高（如 arXiv, ACM DL, Google Scholar, 特定会议 Proceedings）
- **关键词配方**：哪些关键词组合在本方向测试有效，哪些无效
- **搜索策略**：引文追踪、作者追踪、会议扫描、RSS 监控等策略
- **模式发现**：在多次采集中发现的主题聚类、趋势方向、领域结构的模式识别
- **信息源评价**：对不同来源（arXiv, 博客, 正式出版物）的质量和时效性评估

### 示例

```markdown
## 领域调研策略

### 有效数据库
- arXiv cs.HC: 产出不确定，掺水论文偏多，仅可作过滤后的补充
- CHI / UIST / CSCW proceedings: 核心来源，质量稳定
- Google Scholar: 发现跨领域引用线索有效

### 有效关键词
- "human-AI collaboration" + "education": 高产量
- "multi-agent" + "learning": 2025-2026 新热点
- "AI ethics" + "empirical": 产量适中但质量高

### 搜索策略
- 每周一扫描最新 arXiv cs.HC listing
- 会议截稿后 2 周重点扫描 CHI/UIST 相关 track
- 对每篇高分论文做前向/后向引用追踪
```

策略 section 会被 `sync_from_source.py` 自动提取，并聚合到对应角色的成员页面中展示。

## 角色采集范围与质量门槛

### 各角色优先采集方向

| 角色 | 优先采集方向 | 采集原则 |
|------|------------|---------|
| 悠仁 | 跨领域追踪、AI+X 交叉创新 | 鼓励多领域探索 |
| 野蔷薇 | HCI、AI Ethics、批判性研究 | 鼓励批判性视角 |
| 惠 | Learning Analytics、教育AI方法论 | 鼓励方法论深耕 |
| 五条悟 | LLM/Agent/多模态/前沿SOTA | 鼓励追踪最新突破 |

### ML / CV / NLP 质量门槛（重要）

ML、CV、NLP 已高度拥挤，论文产出量大但同质化严重。因此：
- **降低 ML/CV/NLP 基础方向的比例要求**
- **只采集最好的**：顶会 Best Paper / Oral / Outstanding，或者方法有根本性创新的工作
- **跳过标准**：没有显著方法创新的 Incremental work、standard benchmark + 1% improvement 类论文一律跳过
- **如果当月没有值得采集的 ML/CV/NLP 论文，该角色当月可不提交此类论文**

判断标准：
- 是否是 Best Paper / Oral / Outstanding Paper？
- 方法创新是否属于该领域的 "first" 或 "breakthrough"？
- 是否解决了该领域长期存在的 open problem？

### 其他方向（HCI、AI Ethics、Learning Analytics、AI Agent、跨领域应用等）

维持正常采集标准，质量优先但鼓励探索。
