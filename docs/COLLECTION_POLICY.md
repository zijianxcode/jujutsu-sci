# 内容采集规则

## 最小采集单元

采集的最小单元是一篇论文。每篇论文对应一个目录，目录里只有一个完整论文总结：

```text
records/YYYY/MM/DD/HH/<paper-key>/论文总结.md
```

这样可以避免同一篇论文在论文总结、角色长文、团队讨论和升级迭代里重复出现完整摘要。

## 文件职责

`论文总结.md` 是事实源，负责完整记录论文信息和六段式分析。

`<角色>-能力进化.md` 是判断源，负责记录某个角色对这篇论文的评分、批判、关联和下一步动作。

`团队讨论.md`、`深度讨论.md`、`升级迭代.md` 不作为默认采集产物。只有当用户明确要求一次团队讨论或阶段评审时才生成。

## 必填信息

`论文总结.md` 必须写明：

- 论文标题
- 作者
- 来源、会议、期刊、arXiv、DOI 或 URL
- 日期
- 领域标签
- 综合评分

正文保持六段式：

1. 研究什么？
2. 为什么研究？
3. 别人做过什么？
4. 作者怎么研究的？
5. 发现了什么？
6. 价值和不足？

## 去冗余规则

同一篇论文已有 `<paper-key>/论文总结.md` 时，只更新该文件，不新建第二个总结。

角色短评不得复制 `论文总结.md` 的完整内容。它只保留角色自己的判断。

`*-论文总结.md` 这类带角色前缀的文件不再作为新采集格式使用。
