# 版本历史

本文档记录 `jujutsu-sci` 的版本定位、更新说明和发包归档位置。

## v2.1.0 — Agent 能力升级：引用溯源、结构审查与方法论沉淀

对应文件：
- `releases/packages/jujutsu-sci-v2.1.0.zip`
- Git 标签：`v2.1.0`

### 版本定位

v2.1.0 是一次面向 Agent 研究能力的系统性升级。如果说 v2.0 完成了"采什么、怎么存"的基础设施，v2.1 回答的是"采得对不对、写得全不全、方法能不能复用"——即从"搬运工"到"研究者"的关键跃迁。

本次升级的核心命题是：**让 Agent 像一位严谨的研究者一样工作**——写论文总结前验证引用来源、交稿前自检结构完整性、做完一轮调研后沉淀可复用的方法论。

### 三项核心能力

#### 1. 引用溯源校验（`verify_citations.py`）

**研究视角**：一篇论文总结中的 arXiv ID 或 DOI，是读者追溯原始文献的唯一桥梁。一个失效的引用等于研究链条的断裂。

**做了什么**：
- 新增独立的引用验证脚本，自动扫描源目录所有 `论文总结.md`，提取 arXiv ID 和 DOI
- 通过 arXiv API 批量验证 arXiv ID 的真实性（检查论文是否存在、标题是否匹配）
- 通过 CrossRef API 验证 DOI 的有效性
- 支持 `--fix` 模式在 Markdown 末尾追加验证状态注释，支持 `--dry-run` 预览
- 符合学术出版中"引用必须可追溯"的基本规范

**在 Agent 工作流中的位置**：每个采集 Agent 写完论文总结后，自动运行引用校验，确保提交前所有文献链接真实有效。

#### 2. 论文结构完整性审查（`check_paper_quality` / `sync_from_source.py`）

**研究视角**：一篇合格的论文总结，应该像一篇合格的投稿论文一样，结构完整、元数据齐全、评分有效。这不是"排版检查"，而是研究者交稿前的自我审查。

**做了什么**：
- 新增 `check_paper_quality()` 函数，自动检查每篇论文总结的元数据完整性（标题、作者、来源、日期、领域标签、评分）
- 模糊匹配六段式结构（研究什么 / 为什么研究 / 别人做过什么 / 作者怎么研究 / 发现了什么 / 价值和不足），容忍编号前缀和措辞变体
- 检查评分是否有效数值，检测同目录下是否有角色评分
- `sync_from_source.py --check-quality` 在生成页面同时输出质量报告，写入 `quality-report.json`
- 独立脚本 `check_paper_quality.py` 支持 `--json` 输出，可集成到 CI 或其他工具链
- **非阻塞设计**：质量问题打印警告但不中断站点生成，符合"先发布再改进"的工程原则

**在 Agent 工作流中的位置**：每个采集 Agent 在标记完成前，自检清单确保六段齐全、元数据完整、评分有效。

#### 3. 领域调研方法论沉淀（策略层）

**研究视角**：一个好的研究者不只是读论文，而是从每一轮调研中提炼出"怎么找更好"的方法论。哪些数据库在这个方向产出高？哪些关键词组合有效？哪些搜索策略值得复用？这些隐性知识如果不被记录，就会在每次调研中重新试错。

**做了什么**：
- 在 `COLLECTION_POLICY.md` 定义 `## 领域调研策略` 标准格式（有效数据库、关键词配方、搜索策略、模式发现、信息源评价）
- `extract_strategy_section()` 自动从角色能力进化文档中提取策略章节
- `collect_strategies()` 按角色聚合所有调研策略
- 成员页面自动展示该角色积累的方法论知识
- `site.css` 新增策略卡片的视觉样式，与正文形成可区分的"方法论"信息层

**在 Agent 工作流中的位置**：每个采集 Agent 被要求在角色能力进化文档中追加策略章节，把"这次怎么找到的"变成"下次可以怎么找"。

### Agent 工作流重构

v2.1.0 重新编排了 Hermes 采集 Agent 的执行流程，将三项新能力嵌入日常采集链路：

```
采集任务 → 搜索论文 → 生成论文总结 + 角色能力进化（含策略章节）
         → 引用溯源校验（verify_citations.py --fix）
         → 结构完整性自检（六段式 / 元数据 / 评分 checklist）
         → 标记完成
```

六个 Hermes cron 任务的 prompt 均已更新，覆盖三个采集 Agent（悠仁 / 野蔷薇 / 惠）、汇总 Agent（五条悟）、同步 Agent 和周报 Agent。

### 测试覆盖

- `tests/test_verify_citations.py`：24 个测试，覆盖 arXiv ID / DOI 提取、API 响应解析、文件扫描、报告格式和集成场景
- `tests/test_pure_functions.py`：新增 12 个测试，覆盖完整性检查（含模糊匹配）、策略提取和聚合逻辑
- 全部 76 个测试通过，回归确认 `sync_from_source.py` 无参数运行行为不变

### 维护提醒

- `verify_citations.py` 依赖网络访问 arXiv API 和 CrossRef API，离线环境会跳过校验而非报错
- 策略章节是可选的——Agent 只在确实有可复用发现时才写，不强制填充
- `--check-quality` 的 `quality-report.json` 每次运行会覆盖，如需历史追踪请自行归档
- 新增的 `check_paper_quality.py` 和 `verify_citations.py` 是独立脚本，不依赖 `sync_from_source.py` 的其他模块

## 2026-05-15 - 全局中英文切换

### 更新定位

这是面向 `bananabox.plus/academy/` 的全局可访问性与国际化更新。默认仍为中文，右上角提供 `中 / EN` 切换入口，切换偏好会在本地浏览器保存并跨页面生效。

### 主要更新

- 新增 `site-i18n.js`，统一处理首页、排行榜页和详情页的 UI 文案切换。
- `sync_from_source.py` 在所有生成页面注入语言切换入口和 i18n 脚本，避免下次同步覆盖手工改动。
- 英文态覆盖站点外壳、导航、按钮、统计、搜索占位符、排行榜状态、详情页工具栏与动态空状态。
- 论文正文、论文标题和记录标题不做自动翻译，避免破坏研究内容语义。
- 更新测试，确认生成页面包含语言切换入口和全局 i18n 脚本。

### 维护提醒

发布 `academy` 时必须同步 `site-i18n.js`。如果只同步 HTML 和 CSS，线上会出现右上角按钮存在但无法切换语言的问题。

## 2026-04-29 - 信息获取提效与排行榜扩展

### 更新定位

这是 v2.0 之后的信息呈现维护更新，目标是减少首页框线噪音，把高频研究动作前置：先扫资讯、看五条排行榜、进入研究方向，再按需追溯成员判断。

### 主要更新

- 首页品牌统一为 `研究所 / 无量空处`，弱化中二感，同时保留项目特色。
- “核心学术入口”下新增 `今日研究工作台`，把榜首精读、评分补齐和热点追踪变成明确行动入口，减少页面下方空白。
- “近 3 天论文排行榜”首页仅保留前 5，并新增 `More` 进入 `ranking.html`；二级页默认显示前 10，可切换前 20。
- 排行榜前 3 名增加金、银、铜视觉标识和细线强调，让优先级更容易扫读。
- 首页 `研究方向雷达` 和 `研究角色` 恢复统一外框，但内部改为轻量分隔，避免框套框。
- 顶部 `研究所` 与 `当前学术概览` 改为同一行等高布局，解决底部边框不齐的问题。

### 维护提醒

本次新增了 `ranking.html`。以后只要首页排行榜逻辑、CSS 或生成脚本变化，发布时必须同时同步新增/变化的 HTML 与 `site.css`，否则 CloudBase 可能出现首页有入口但二级页缺失、或线上仍显示旧布局的问题。

## v2.0.0 - Hermes 迁移与 IA 2.0

对应文件：
- `releases/packages/jujutsu-sci-v2.0.0.zip`
- Git 标签：`v2.0.0`

### 版本定位

v2.0.0 是当前学术站点的信息架构升级版本，重点是把 Hermes 采集链路、研究包入口、首页学术视图和详情页阅读体验整理成更稳定的发布基线。

### 主要更新

- 首页收敛为学术协作入口，突出论文追踪、研究方向、成员进展和高星论文。
- 详情页阅读体验升级，改进侧边导航、目录切换、搜索、Obsidian 导出和移动端阅读。
- 增加 IA 迭代和 Hermes 迁移文档，明确内容从采集到生成、发布、镜像的完整链路。
- 更新 `sync_from_source.py` 和测试，适配新的内容生成规则与页面结构。
- 补充 `docs/assets/` 下的页面截图，作为 IA 2.0 和详情页改版的视觉记录。

### 发包差异

相对 v1.0.0：
- 新增 12 个 `docs/assets/` 截图文件。
- 更新 22 个既有文件，覆盖页面产物、样式、详情页脚本、生成脚本、测试与 IA 文档。

### 发布说明

该版本已经适合直接作为静态站点发包留档。若发布到国内访问入口，需要继续按 `docs/DEPLOYMENT.md` 中的 CloudBase 子站链路执行：

`jujutsu-sci` 生成结果 -> `personal-homepage/academy/` 镜像 -> CloudBase 静态托管部署。
