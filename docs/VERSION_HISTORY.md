# 版本历史

本文档记录 `jujutsu-sci` 的版本定位、更新说明和发包归档位置。

## 2026-04-29 - 信息获取提效与排行榜扩展

### 更新定位

这是 v2.0 之后的信息呈现维护更新，目标是减少首页框线噪音，把高频研究动作前置：先扫资讯、看五条排行榜、进入研究方向，再按需追溯成员判断。

### 主要更新

- 首页品牌统一为 `研究所 / 无量空处`，弱化中二感，同时保留项目特色。
- “核心学术入口”下新增 `今日研究工作台`，把榜首精读、评分补齐和热点追踪变成明确行动入口，减少页面下方空白。
- “近 3 天论文排行榜”首页仅保留前 5，并新增 `More` 进入 `ranking.html`；二级页默认显示前 10，可切换前 20。
- 排行榜前 3 名增加金、银、铜视觉标识和细线强调，让优先级更容易扫读。
- 首页 `研究方向雷达` 和 `研究角色` 恢复统一外框，但内部改为轻量分隔，避免框套框。
- 顶部 `研究所` 与 `当前学术概览` 改为同一行等高布局，解决底部边框不齐的问题。

### 维护提醒

本次新增了 `ranking.html`。以后只要首页排行榜逻辑、CSS 或生成脚本变化，发布时必须同时同步新增/变化的 HTML 与 `site.css`，否则 CloudBase 可能出现首页有入口但二级页缺失、或线上仍显示旧布局的问题。

## 2026-04-28 - v2.0 后维护更新

对应提交：
- `550e1ab Improve homepage UX and ten point rankings`

### 更新定位

这是 v2.0.0 之后的维护更新，重点解决首页阅读密度、10 分制评分规则和 CloudBase 发布链路的一致性问题。

### 主要更新

- 首页减少过多卡片边框，压缩低频信息，让“核心学术入口”和“近 3 天论文排行榜”更适合快速扫读。
- 排行榜固定显示前 5 篇，卡片去掉低价值来源路径和重复说明，优先保留标题、日期、评分、下一步动作和阅读入口。
- 新采集评分展示统一为 10 分制；旧 5 分制评分只作为兼容输入，前端仍转换为 `/10` 展示。
- 支持从 `五条悟-汇总.md` 读取每日评分，并根据 `yujin-7218` 这类研究包 key 回填到首页排行榜。
- 补充回归测试，避免以后“汇总里评了分但首页仍显示未评分”。
- 同步 `personal-homepage/academy/` 并部署 CloudBase，确认 `bananabox.plus/academy/` 已显示新评分。

### 维护提醒

以后凡是修改首页 UI、评分展示或详情页交互，发布 `academy` 时不能只同步 HTML，还要同步 `site.css`、`site-detail.js`、`site-index.js` 和相关图片资源。

## v2.0.0 - Hermes 迁移与 IA 2.0

对应文件：
- `releases/packages/jujutsu-sci-v2.0.0.zip`
- Git 标签：`v2.0.0`

### 版本定位

v2.0.0 是当前学术站点的信息架构升级版本，重点是把 Hermes 采集链路、研究包入口、首页学术视图和详情页阅读体验整理成更稳定的发布基线。

### 主要更新

- 首页收敛为学术协作入口，突出论文追踪、研究方向、成员进展和高星论文。
- 详情页阅读体验升级，改进侧边导航、目录切换、搜索、Obsidian 导出和移动端阅读。
- 增加 IA 迭代和 Hermes 迁移文档，明确内容从采集到生成、发布、镜像的完整链路。
- 更新 `sync_from_source.py` 和测试，适配新的内容生成规则与页面结构。
- 补充 `docs/assets/` 下的页面截图，作为 IA 2.0 和详情页改版的视觉记录。

### 发包差异

相对 v1.0.0：
- 新增 12 个 `docs/assets/` 截图文件。
- 更新 22 个既有文件，覆盖页面产物、样式、详情页脚本、生成脚本、测试与 IA 文档。

### 发布说明

该版本已经适合直接作为静态站点发包留档。若发布到国内访问入口，需要继续按 `docs/DEPLOYMENT.md` 中的 CloudBase 子站链路执行：

`jujutsu-sci` 生成结果 -> `personal-homepage/academy/` 镜像 -> CloudBase 静态托管部署。

## v1.0.0 - GitHub 源同步基线

对应文件：
- `releases/packages/jujutsu-sci-v1.0.0.zip`
- Git 标签：`v1.0.0`

### 版本定位

v1.0.0 是从 GitHub 源同步后整理出的第一个可归档静态站点版本，用于保留早期页面结构、自动同步脚本和基础部署说明。

### 主要更新

- 固化静态页面结构，包括首页、论文、归档、主题、成员、讨论、升级迭代和高星论文。
- 保留 `auto_sync_site.sh`，记录学术站源码仓库与 `personal-homepage/academy/` 镜像之间的同步关系。
- 增加内容同步、部署发布、问题日志和项目结构文档，方便后续维护。
- 提供基础测试，覆盖源内容同步生成的关键行为。

### 发布说明

该版本可用于回滚到早期静态站点形态。由于后续国内访问依赖 CloudBase / `academy` 镜像链路，回滚时不能只替换 `jujutsu-sci`，还需要同步 `personal-homepage/academy/` 并重新发布。
